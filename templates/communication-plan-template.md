## 🎯 WHEN TO USE COMMUNICATION PLAN

| Use Communication Plan When | Don't Use When |
|-----------------------------|----------------|
| ✓ Project has 3+ stakeholder groups | ✗ Single-team internal project |
| ✓ Distributed or cross-functional teams | ✗ Communication already ad hoc and working |
| ✓ Executive visibility required | ✗ Short spike / proof-of-concept only |
| ✓ Change management is part of delivery | ✗ Stakeholders are co-located daily |
| ✓ Regulatory or audit trail needed | ✗ Agile team with daily standups (use RACI) |

## ✅ COMMUNICATION PLAN QUALITY CHECKLIST

```
☐ All stakeholder groups identified in matrix
☐ Information need defined per stakeholder (not generic)
☐ Frequency appropriate to stakeholder influence/interest
☐ Channel matches stakeholder preference and formality
☐ Each communication has a named owner
☐ Status report RAG criteria defined (what triggers Amber/Red)
☐ Meeting minutes distributed within 24h of meeting
☐ Action items have single owner and firm due date
☐ Escalation path tested and contacts confirmed
☐ Decision log maintained throughout project
☐ Communication plan reviewed at each phase gate
```
